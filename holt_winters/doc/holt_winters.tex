\section{Introduction}
\label{sec:intro}

This package implements the Holt--Winters forecasting method, namely
triple exponential smoothing with level, trend and seasonal components.
It requires time-series data with a seasonal dimension. Quarterly or
monthly data are the best bet for suitability. Daily or hourly data
might be suitable, subject to the qualifications noted in
section~\ref{sec:daily}.

There are two variants of Holt--Winters, differing with respect to their
treatment of the seasonal component, which can be either additive or
multiplicative. The respective filtering recursions and forecasting
equations are shown below. The notation is based on \cite{hyndman2021}:
$\ell$ denotes the level, $b$ the trend or slope, and $s$ the seasonal.
In addition $m$ denotes the periodicity of the data (gretl's \verb|$pd|)
and $k$ is the integer part of $(h-1)/m$ ($=0$ when when $h=1$).

Additive:
\begin{align*}
  \ell_t &= \alpha (y_t - s_{t-m}) + (1-\alpha) (\ell_{t-1} + b_{t-1})\\
  b_t &= \beta (\ell_t - \ell_{t-1}) + (1-\beta) b_{t-1}\\
  s_t &= \gamma (y_t - \ell_t) + (1-\gamma) s_{t-m}\\
  \hat{y}_{t+h|t} &= \ell_t + hb_t + s_{t+h-m(k+1)}
\end{align*}

Multiplicative:
\begin{align*}
  \ell_t &= \alpha (y_t / s_{t-m}) + (1-\alpha) (\ell_{t-1} + b_{t-1})\\
  b_t &= \beta (\ell_t - \ell_{t-1}) + (1-\beta) b_{t-1}\\
  s_t &= \gamma\, y_t / \ell_t + (1-\gamma) s_{t-m}\\
  \hat{y}_{t+h|t} &= (\ell_t + hb_t)\, s_{t+h-m(k+1)}
\end{align*}

It may be helpful to start with an overview. Like most functions in
gretl, \dtk{holt_winters} confines its operations to the sample range
in force when the function is called. We'll refer to the (1-based)
start and end of this range as $t_1$ and
$t_2$, respectively.\footnote{This range can be set via the
  \texttt{smpl} command. If \texttt{smpl} has not been invoked since a
  dataset was opened $t_1$ will equal 1 and
  $t_2$ will equal the total number of observations.} The package will
operate on one or two contiguous sub-ranges:
\begin{enumerate}
\item a training or estimation range, starting at
  $t_1$, where the Holt--Winters recursion is applied and
  one-step-ahead forecasts ($h=1$) are computed; and
\item an `out-of-sample' range (if applicable), ending at
  $t_2$, where forecasts with
  $h=1,2,...,n$ are computed, conditional on the state of information
  at the end of the first range. The value of
  $n$ here is determined by the argument \dtk{n_fcast} (see below).
\end{enumerate}

Note that the number of training observations is a residual quantity: it
equals $t_2 - t_1 + 1 - n$. There must be no missing values for the
target series in that range. It's OK if there are missing values in the
second range, and in fact if there any are non-missing values they will
be ignored. If you just want to use Holt--Winters as a
filter---decomposing a series into its components---and don't care about
out-of-sample forecasts, it's OK to have $n=0$. Then the function will
try to ensure that any trailing missing values of the input series are
excluded.

\section{The public function}

The \dtk{holt_winters} function has the following signature:
\begin{code}
function bundle holt_winters (const series y, const bundle specs)
\end{code}

The required argument \texttt{y} specifies the series for filtering and
forecasting. The second (optional) bundle argument may contain one or
more of the following options:

\begin{itemize}
\item \texttt{theta} (matrix): if supplied, this must be a 3-vector
  holding the smoothing parameters $\theta$ = ($\alpha$, $\beta$,
  $\gamma$), with $0 \le \theta_i \le 1,\, \forall i$.  If
  \texttt{parms} is not given these parameters are determined
  automatically.
\item \texttt{init} (matrix): if supplied, this must be a vector of
  length $2+m$ holding initial values. Let $\tau$ indicate the first
  observation used in the recursion; then the \texttt{init} values
  should be $\ell_{\tau-1}$, $b_{\tau-1}$, and
  $s_{\tau-i}, i = m, m-1, \dots,1$. If \texttt{init} is not given the
  initializer is set automatically.
\item \texttt{method} (integer): specifies the treatment of the seasonal
  term as either additive (1) or multiplicative (2, the default).
\item \dtk{n_fcast} (integer): specifies the number of observations at
  the end of the incoming sample range to reserve for out-of-sample
  forecasting. The default value is 0 (filtering, with no out-of-sample
  forecast).
\item \texttt{verbose} (integer): controls the verbosity of output, with
  levels 0, 1 (the default), 2 or 3.
\end{itemize}

\subsection{Return value}

This is a bundle containing the following elements:
\begin{itemize}
\item \texttt{theta}: a 3-vector holding the parameter values used in
  forecasting.
\item \texttt{init}: a ($2+m$)-vector holding the initializer used in
  forecasting.
\item \texttt{method}: 1 if the additive seasonal method was used,
  otherwise 2.
\item \texttt{X}: a $T \times 5$ matrix.  Column 1 holds the data
  passed as \texttt{y}; columns 2 to 4 the estimated level, trend and
  seasonal indices; and column 5 holds forecasts. More details on this
  matrix are provided below.
\item \texttt{RMSE}: the root mean squared error for the 1-step ahead
  forecasts made using the training data.
\end{itemize}

\section{Command-line usage}
\label{sec:auto}

We'll start with a simple example---an approximate but passable
replication of the multiplicative Australian tourism example given by
\cite{hyndman2021}. Here's the hansl code:\label{pg:ex1}
\begin{code}
open aus_holidays.gdt --frompkg=holt_winters
dataset addobs 12
bundle hwb = holt_winters(Trips, _(n_fcast=12))
\end{code}

The \dtk{aus_holidays} dataset contains 80 quarterly observations from
1998:1 to 2017:4, which will be used for training the model. The twelve
added observations, 2018:1 to 2020:4, will then be used for
out-of-sample forecasting. Since values are not specified for
\texttt{theta} or \texttt{init}, both the parameters and the initializer
for the Holt--Winters recursion will be optimized automatically---see
section~\ref{sec:details} for details.

There being no call to \texttt{smpl} in the script above, $t_1$ and
$t_2$ will be at their default values: in context, 1 and 92.  So the
value of 12 for \dtk{n_fcast} selects just the added `empty'
observations, giving a genuine out-of-sample forecast. In some cases one
might want a `pretend' out-of-sample forecast---a `forecast' of values
that are present in the dataset but not used in estimation. With the
Australian tourism data one could do something like the following
\begin{code}
open aus_holidays.gdt --frompkg=holt_winters
bundle hwb = holt_winters(Trips, _(n_fcast=8))
\end{code}
to get pretend out-of-sample forecasts for 2016:1 to 2017:4.

\section{Technical details}
\label{sec:details}

This section offers some details on the rule-of-thumb initialization
with which the Holt--Winters recursion is started, and the optimization
that is carried out if \texttt{init} and/or \texttt{theta} are not given
by the caller.

First, when \texttt{init} is not supplied, the initializer is calculated
as per the suggestion in \cite{hyndman98} (also see \cite{hyndman2010}:
\begin{itemize}
\item The initial level, $\ell_{\tau-1}$, is set to the mean of the
  $m$ values $y_{\tau}, y_{\tau+1},\dots, y_{\tau+m-1}$.
\item The initial trend, $b_{\tau-1}$, is set to the mean of the
  $m$ seasonal differences
 \[
    y_{\tau+i+m} - y_{\tau+i},\; i=0,1,\dots,m-1
 \]
\item The initial seasonals are then set to the ratio of period-specific
  $y_t$ to $\ell_{\tau-1}$ for the first $m$ observations
  (multiplicative case) or the difference between $y_t$ and
  $\ell_{\tau-1}$ (additive case).
\end{itemize}

As regards optimization, three cases arise.
\begin{itemize}
\item If neither \texttt{theta} nor \texttt{init} is given, the
  parameter values and initializer are optimized jointly, with the
  criterion of minimizing the sum of squared forecast errors for the
  training range. The starting point for the parameters is
  $\alpha = \beta = 0.2, \gamma = 0.05$; the starting point for the
  initializer is as described above. Optimization is performed using the
  L-BFGS algorithm (see the chapter on Numerical methods in the \GUG\
  for details), with $\alpha$, $\beta$ and $\gamma$ constrained to lie
  in $[0,1]$.
\item If an initializer is given by the caller but \texttt{theta} is not
  given, then $\theta$ is optimized conditional on the supplied
  \texttt{init}.
\item If \texttt{theta} is given but not \texttt{init}, the initializer
  is just the rule-of-thumb value; no optimization is performed. In
  principle the initializer could be optimized conditional on $\theta$
  and that's a possible future refinement if it seems there's a real use
  case.
\end{itemize}

\section{Daily and hourly data}
\label{sec:daily}

As mentioned in section~\ref{sec:intro}, the Holt--Winters procedure may
be applicable to daily data, with $m$ = 5, 6 or 7 days per
week. In this context `seasonality' means intra-week
variability. However, the results will be meaningful only if the data
are `complete', in the sense that each week in the sample range includes
$m$ valid daily observations. It should go without saying that financial
market data, even if complete, are unlikely to be suitable since
seasonal variation would be a clear violation of market efficiency (in
the narrow sense in which market efficiency is a plausible hypothesis).

Gretl also supports hourly data ($m=24$) and Holt--Winters may work if
there's `seasonality' in the form of intra-day variability. Again,
however, the series in question has to be complete over the given sample
range.
